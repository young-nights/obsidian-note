# MOC Template 使用说明

## 什么是 MOC

MOC（Map of Content）是一种知识组织方式，由 Nick Milo 提出。它不是传统的文件夹分类，也不是严格的层级目录，而是一张**内容地图**——通过一个中心页面，将散落在不同位置的相关笔记以链接的方式串联起来。

与文件夹分类的本质区别：
- **文件夹**：一条笔记只能属于一个位置
- **MOC**：一条笔记可以出现在多张地图中，按主题、上下文自由关联

本模板基于 **Templater** + **Dataview** 插件，一键生成带有动态查询能力的 MOC 页面。

> [!info] 适用场景
> 任何需要围绕一个主题整理多篇笔记的场景：学习路线、项目文档、研究专题等。

## 模板核心功能一览

| 区块 | 说明 | 动态/静态 |
|------|------|-----------|
| 📋 概览 | 显示 MOC 状态、文件夹路径、笔记数、创建时间 | 动态（DataviewJS） |
| ⭐ 核心笔记 | 手动标记最重要的 3~5 篇 | 静态 |
| 🗂️ 分类导航 | 创建时自定义分类，每个分类下手动整理 | 静态（Templater 生成骨架） |
| 🔗 关联 MOC | 链接其他主题的 MOC，构建知识网络 | 静态 |
| 📌 待办 & 下一步 | 记录该主题的未完成事项 | 静态 |
| 📚 学习路径 | 自动列出文件夹内所有笔记，按创建时间排序 | 动态 |
| 🕐 最近修改（7天内） | 自动展示最近 7 天修改的笔记（最多 10 条） | 动态 |
| 📊 全部笔记 | 自动列出文件夹内所有笔记，按修改时间倒序 | 动态 |
| 🏷️ 按标签聚合 | 自动按 tag 分组统计，显示每组笔记数 | 动态 |
| 🐛 问题日志 | 手动记录踩坑和解决方案 | 静态 |

## 使用步骤

### 1. 前置条件

确保已安装并启用以下 Obsidian 插件：
- **Templater**（模板引擎）
- **Dataview**（数据查询）

### 2. 创建新的 MOC

1. 在目标文件夹中创建一个空白笔记（名称随意）
2. 打开命令面板（`Ctrl/Cmd + P`），运行 `Templater: Open Insert Template Modal`
3. 选择 `MOC Template`
4. 按提示依次填写：
   - **MOC 名称**：默认取当前文件夹名，可修改
   - **Emoji**：用于标识的图标，默认 📁
   - **状态**：`planning` / `active` / `archived`
   - **自定义分类**：逗号分隔，回车跳过则使用默认四类
5. 模板自动完成：
   - 重命名文件为 `📁 名称 MOC`（按你填的 emoji 和名称）
   - 生成完整页面内容

> [!tip] 快捷方式
> 可以在 Templater 设置中为该模板配置 Hotkey，实现一键创建 MOC。

## 各区块详细说明

### 📋 概览

使用 DataviewJS 从当前笔记的 frontmatter 中提取信息，展示为表格：

| 属性 | 来源 |
|------|------|
| 状态 | frontmatter `status` 字段，映射为 🟢/🟡/📦 |
| 文件夹 | 当前笔记所在路径 |
| 笔记数 | 当前笔记的出链数量 |
| 创建时间 | 笔记创建时间 |

### ⭐ 核心笔记

手动维护。放入该主题下最常查阅、最重要的 3~5 篇笔记链接。这是打开 MOC 时第一眼看到的内容。

### 🗂️ 分类导航

创建时输入的分类（或默认四类）会生成对应的三级标题。在每个分类下手动整理笔记链接。默认分类：

- **核心笔记** — 最重要的内容
- **环境与配置** — 环境搭建、配置记录
- **问题与排错** — 踩坑记录
- **参考资源** — 外部资料、文档链接

### 📚 学习路径

自动查询当前文件夹内所有 `.md` 文件（排除自身和含 `moc` tag 的笔记），按创建时间正序排列。

- 有 `基础`/`进阶`/`高级` tag 的笔记会显示对应难度图标
- 其余显示 📝

### 🕐 最近修改（7 天内）

自动查询 7 天内修改过的笔记，最多显示 10 条，按修改时间倒序。适合快速回顾近期工作。

### 📊 全部笔记

列出文件夹内所有笔记，显示图标、链接、标签、出链数、修改时间。可自定义图标映射（见下文）。

### 🏷️ 按标签聚合

遍历文件夹内所有笔记的 tags，按标签分组统计。按笔记数量降序排列，一眼看出哪些标签覆盖最多内容。

## DataviewJS 查询逻辑说明

模板中所有动态查询都使用：

```javascript
dv.pages('"<folderPath>"')
```

而不是 `dv.pages('""')`。原因：

| 写法 | 含义 | 适用场景 |
|------|------|----------|
| `dv.pages('"folderPath"')` | 只查指定文件夹下的笔记 | ✅ MOC 场景：只关心当前主题相关笔记 |
| `dv.pages('""')` | 查全库所有笔记 | ❌ 会混入不相关内容，噪音太大 |

> [!warning] 路径绑定
> 模板在生成时将 `folderPath` 硬编码写入 DataviewJS 代码块。如果之后移动了文件夹位置，查询路径不会自动更新，需要手动修改。

模板还做了进一步过滤：

```javascript
.where(p =>
  p.file.ext === "md"                          // 只要 Markdown 文件
  && p.file.name !== dv.current().file.name    // 排除 MOC 自身
  && !p.file.path.includes("Templates")        // 排除模板文件
  && !p.file.tags.includes("moc")              // 排除其他 MOC 文件
)
```

## 自定义指南

### 修改默认分类

在模板弹窗提示「自定义分类」时，输入逗号分隔的分类名即可。例如：

```
基础概念, 实战笔记, 源码分析, 常见问题
```

如果想永久修改默认值，编辑 `MOC Template.md` 中的：

```javascript
["核心笔记", "环境与配置", "问题与排错", "参考资源"]
```

### 调整「最近修改」天数

找到 `🕐 最近修改` 区块，修改：

```javascript
dv.date("today").minus(dv.duration("7 days"))
//                                       ↑ 改天数
```

### 自定义笔记图标

在 `📊 全部笔记` 区块中修改 `iconMap`：

```javascript
const iconMap = { issue: "🐛", config: "⚙️", guide: "📖" };
// ↑ 添加更多映射，key 对应笔记的 tag 名
```

### 调整学习路径的难度标签

在 `📚 学习路径` 区块中修改 `levelMap`：

```javascript
const levelMap = { "基础": "📗 基础", "进阶": "📘 进阶", "高级": "📕 高级" };
// key 是笔记的 tag 名，value 是显示文本
```

## 注意事项

> [!warning] 插件依赖
> 本模板依赖 **Templater** 和 **Dataview** 两个社区插件。未安装或未启用会导致模板插入失败或动态区块显示为空。

- **Templater** 用于弹窗交互、自动重命名、条件生成分类骨架
- **Dataview** 用于所有 ` ```dataviewjs ``` ` 代码块的动态查询

> [!tip] 移动文件夹后
> DataviewJS 中的查询路径是模板生成时硬编码的。如果移动了文件夹，需要手动更新各代码块中的路径字符串。

> [!info] 模板自动重命名
> 模板执行时会自动将笔记重命名为 `emoji + 名称 + MOC` 格式。如果笔记已有重要内容（非从空笔记开始），重命名不会丢失内容，但文件名会变。
