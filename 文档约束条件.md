# 文档约束条件

> 本文档定义了 Obsidian 笔记库的统一排版规范和约束条件。

---

## 一、字体大小规范

| 元素 | 标签格式 | 示例 |
|------|---------|------|
| 一级标题 | `# <font size=4>标题</font>` | `# <font size=4>IndexTTS 本地部署文档</font>` |
| 二级标题 | `## <font size=3>标题</font>` | `## <font size=3>一、安装 ffmpeg</font>` |
| 三级标题 | `### <font size=2>标题</font>` | `### <font size=2>1.1 访问官网</font>` |
| 正文内容 | `<font size=2>...</font>` 包裹 | 见下方示例 |

**正文包裹示例：**

```markdown
## <font size=3>二、安装 Miniconda</font>

<font size=2>

这里是正文内容，包括文字、代码块、表格、图片等全部包裹在内。

```bash
conda install xxx
```

</font>
```

**约束：**
- `<font size=2>` 开始标签后必须空一行再写正文，且前一行也必须空一行
- `</font>` 结束标签前一行留空，且后一行也留空
- 代码块、表格、图片等全部在 `<font size=2>` 包裹范围内

> [!tip] 纯文档类笔记
> 对于技术说明文档（如 `Templates For Templator/` 下的使用说明），可以不加 `<font>` 标签，直接使用标准 Markdown 格式。`<font>` 包裹主要用于需要精细控制字体大小的部署/教程类笔记。

## 二、Frontmatter 规范

### MOC 文件

```yaml
---
tags: [moc, index, 主题-slug]
created: 2026-04-01 17:02:00
type: MOC
status: active
folder: "文件夹名称"
---
```

**约束：**
- `status` 只能用原始值：`active` / `planning` / `archived`，禁止用 emoji
- 不加 `modified` 字段（使用 Dataview 的 `file.mtime` 自动获取）
- `type` 固定为 `MOC`
- `tags` 第一个固定为 `moc`，第二个为 `index`

### 普通笔记

```yaml
---
tags: [config]
created: 2026-04-01 17:02:00
---
```

**标签使用规范：**

| 标签 | 适用场景 |
|------|---------|
| `config` | 配置类笔记 |
| `guide` | 教程/步骤类 |
| `issue` | 问题记录/踩坑 |
| `基础` / `进阶` / `高级` | 学习类笔记难度标记 |

## 三、提示框规范

使用 Obsidian Callout 语法，禁止使用 `<div style>` 自定义样式框：

```markdown
> [!info] 信息提示
> 这是一段信息提示内容

> [!tip] 小贴士
> 这是一段小贴士内容

> [!warning] 注意
> 这是一段警告内容

> [!caution] 危险
> 这是一段危险警告
```

## 四、Dataview 规范

**必须使用 `dataviewjs`（JavaScript API），禁止使用 `dataview`（DQL）。**

原因：Dataview 0.5.68 的 DQL `choice()` 函数嵌套有兼容性 bug。

```markdown
<!-- ✅ 正确 -->
```dataviewjs
const notes = dv.pages('"文件夹名"')
  .where(p => p.file.ext === "md" && !p.file.tags.includes("moc"))
  .sort(p => p.file.mtime, "desc");

dv.table(["笔记", "修改时间"], notes.map(p => [
  p.file.link,
  p.file.mtime.toFormat("DD HH:mm")
]));
`` `

<!-- ❌ 错误 -->
```dataview
TABLE file.link AS "笔记"
FROM "文件夹名"
`` `
```

### 查询范围规则

> [!warning] 重要：非根目录 MOC 禁止使用全库查询
> 子 MOC 的 DataviewJS 查询必须限定到当前文件夹，禁止使用 `dv.pages('""')`。

| MOC 位置 | 正确写法 | 错误写法 |
|----------|---------|---------|
| 根目录 MOC | `dv.pages('""')` 全库查询 | — |
| 子文件夹 MOC | `dv.pages('"文件夹名"')` | `dv.pages('""')` |

原因：子 MOC 使用全库查询会导致「最近修改」混入其他主题的笔记，「按标签聚合」也会污染标签列表。根目录 MOC 作为总入口，使用全库查询是合理的。

**日期格式：**
- 最近修改列：`DD HH:mm`（如 `01 14:10`）
- 学习路径列：`DD`（如 `01`）
- 全部笔记列：`DD HH:mm`
- 禁止使用 `YYYY-MM-DD`（年份前缀多余）
- 禁止使用 `MM-DD`（月份前缀多余）

**`choice()` 函数：**
- DataviewJS 中可用原生 JS `if/else` 或对象映射替代
- 状态映射使用对象：`const statusMap = { active: "🟢 进行中", planning: "🟡 规划中", archived: "📦 已归档" }`

## 五、MOC 文件结构

每个 MOC 文件的标准结构：

```markdown
---
tags: [moc, index, slug]
created: YYYY-MM-DD HH:mm:ss
type: MOC
status: active
folder: "文件夹名"
---

# 📁 主题名称 MOC

> 一句话描述

## 📋 概览（DataviewJS 动态读取 frontmatter）

## ⭐ 核心笔记（手动维护，3~5 篇）

## 🗂️ 分类导航（手动维护或自定义）

## 🔗 关联 MOC（手动维护，双链 `[[...]]`）

## 📌 待办 & 下一步

## 📚 学习路径（DataviewJS，按创建时间排序）

## 🕐 最近修改（DataviewJS，7 天内）

## 📊 全部笔记（DataviewJS）

## 🏷️ 按标签聚合（DataviewJS，可选）

## 🐛 问题日志（可选）
```

## 六、根级 MOC（总入口）

根目录下创建 `📚 Obsidian Note MOC.md`，主题地图使用 DataviewJS 自动查询所有 `type: MOC` 的文件，无需手动维护。

---

## 七、文件命名规范

- MOC 文件：`emoji + 主题名 + MOC.md`（如 `🎙️ IndexTTS-2 MOC.md`）
- 普通笔记：描述性中文名称（如 `Windows安装FFmpeg.md`）
- 图片文件夹：每个主题文件夹下创建 `images/` 子目录
